# Морфология: перевод на русский (по плану с Gemini)

## Что сделано сейчас

- **Язык по браузеру:** сайт смотрит язык браузера (`navigator.language`). Если украинский (uk) — показываются поля `morphology_stem_uk`, `morphology_spines_uk` и т.д., если русский (ru) — `morphology_stem_ru` и т.д., иначе английский из Wikipedia. Переход на белую страницу Google убран: всё остаётся на тёмном сайте.
- В карточке вида блок **«Морфология»** показывает текст в выбранном языке, если он есть в данных. Поддерживаются суффиксы `_ru` и `_uk`; если перевода нет — показывается английский (`morphology_stem` и т.д.).

## Как потом добавить русский текст в данные (пакетный перевод)

1. **Поля в JSON:** для каждого вида в `data/species/*.json` можно добавить поля `morphology_stem_ru`, `morphology_spines_ru`, `morphology_flower_ru`, `morphology_fruit_ru`. Сайт уже умеет их показывать в приоритете над английским.

2. **Скрипт перевода (когда будет время):**
   - Обходить все `data/species/*.json`, для каждого вида смотреть `morphology_stem` (и др.) без суффикса `_ru`/`_uk`.
   - Отправлять текст в сервис перевода (например [LibreTranslate](https://libretranslate.com/) — бесплатный API, или Google Cloud Translation при наличии ключа).
   - Записывать результат в `morphology_stem_ru`, `morphology_stem_uk` и т.д., сохранять JSON.
   - Делать паузу между запросами, чтобы не перегружать API.
   - После скрипта — вычитка ключевых видов вручную.

3. **Ручной перевод:** при добавлении или правке вида можно сразу заполнять `*_ru` и при необходимости `*_uk` — на сайте будет показан вариант по языку браузера.
