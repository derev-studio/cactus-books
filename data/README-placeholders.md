# Как заполнить карточки (роды и виды)

## Что такое «автоматическая база»

Твоя «база» — это уже существующие файлы:

- **`data/taxonomy.json`** — дерево до родов; у каждого рода поле **`info`** (текст карточки).
- **`data/species/*.json`** — по одному файлу на род; в каждом массиве у вида поле **`description`** (текст карточки).

Заполнить их можно тремя способами.

---

## 1. Ничего не делать

На сайте **пустых карточек нет**: если `info` или `description` пустые, подставляется заготовка («Mammillaria — род кактусов…» и т.п.). Дополнительная база не нужна.

---

## 2. Один раз подставить заготовки в файлы (без нейросети)

Запусти скрипт (нужен Node.js):

```bash
node data/fill-placeholders.js
```

Он пройдёт по всем родам и видам и впишет в пустые поля один и тот же шаблонный текст. После этого можно вручную править только нужные карточки.

---

## 3. Автозаполнение нейросетью (OpenRouter)

Чтобы **все** карточки получили осмысленные короткие описания без ручного ввода:

1. Заведи ключ на [OpenRouter](https://openrouter.ai/keys).
2. В терминале:

```bash
export OPENROUTER_API_KEY=sk-or-...
node scripts/fill-descriptions-ai.js
```

Скрипт по очереди отправит в нейросеть каждый род и каждый вид, получит 1–2 предложения и **запишет их в те же JSON** (`taxonomy.json` и `data/species/*.json`). Заполняются только те поля, где сейчас пусто или стоит заготовка; уже заполненные не перезаписываются.

После одного запуска все карточки будут с текстом. Это и есть «автоматическая база»: не отдельная база, а твои JSON-файлы, заполненные скриптом через API.

Проверить, что будет заполнено, без записи в файлы:

```bash
DRY_RUN=1 node scripts/fill-descriptions-ai.js
```
