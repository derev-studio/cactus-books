# Архив GBIF — большая систематика

Сюда кладём выгрузку с GBIF.org (Darwin Core Archive), чтобы разбирать её по шагам и не смешивать со старой систематикой на сайте.

---

## Куда положить папку

**Вариант (рекомендуемый):** скопируй **всю папку** `0026914-260226173443078` **внутрь этой папки**:

```
cactus/data/sources/gbif/0026914-260226173443078/
```

В итоге путь будет такой:
- `cactus/data/sources/gbif/0026914-260226173443078/occurrence.txt`
- `cactus/data/sources/gbif/0026914-260226173443078/meta.xml`
- и остальные файлы (metadata.xml, verbatim.txt, citations.txt, multimedia.txt, rights.txt, dataset/).

Так всё «кактусное» и систематика лежат в одном проекте, и мы с ними работаем по цепочке.

**Старую систематику не трогаем:** `data/taxonomy.json` и `data/species/*.json` остаются как есть. GBIF — отдельный источник, будем подключать по шагам.

---

## Цепочка действий (по шагам)

Сделаем по одному звену, без спешки.

- **Звено 1.** Разместить архив — скопировать папку `0026914-260226173443078` в `data/sources/gbif/` (как выше). Проверить, что внутри есть `occurrence.txt`, `meta.xml`.
- **Звено 2.** Посмотреть, что внутри — не открывать тяжёлый файл в редакторе. Запустить скрипт: `python3 scripts/inspect_gbif_occurrence.py` (он читает файл построчно, без загрузки 50+ МБ в память). Он покажет размер, число строк, заголовок и первые записи. Ничего не меняем, только смотрим.
- **Звено 3.** Связать с нашей систематикой — решить, как данные GBIF показывать на сайте: как справочник названий/синонимов или как часть дерева. Старое дерево (Бакеберг) не заменяем, добавляем источник и при необходимости соответствие «GBIF ↔ наше дерево».
- **Звено 4.** Подготовить данные для сайта — из архива учёных получить то, что нужно для отображения: список видов (род + вид), при желании синонимы из meta.xml, чтобы связать с нашим деревом и карточками.
- **Звено 5.** Разложить на сайте — оформить так, чтобы систематика из GBIF легла на сайт красиво: дерево, карточки видов, подписи «есть в GBIF» или ссылки. Делать по одному подсемейству или роду, без спешки.
- **Подвиды, разновидности, формы** — из архива вытаскиваются не только виды, но и инфравидовые таксоны (subspecies, variety, form). Они добавляются в карточку родительского вида скриптом `scripts/gbif_add_infraspecific.py`, чтобы в одной карточке было видно все близкородственные таксоны. Ничего из выгрузки не теряем.

Сейчас делаем **звено 1**: ты копируешь папку в `data/sources/gbif/`. Когда скопируешь — напиши, перейдём к звену 2 (посмотрим, что в `occurrence.txt`).

---

## Тяжёлый файл (50+ МБ)

`occurrence.txt` большой — не открываем его целиком в памяти. Работаем **постепенно**:
- просмотр: только первые строки (заголовок + несколько записей);
- подсчёт строк: построчно, без загрузки всего файла;
- подготовка данных для сайта: скрипты читают файл построчно или кусками, без загрузки всего в память.
Так не будет тормозов и ошибок из‑за нехватки памяти.
