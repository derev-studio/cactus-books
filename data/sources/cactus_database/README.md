# Папка cactus_database — разрозненные источники (Википедия, WFO, образцы)

Файлы собраны из разных мест (в т.ч. с помощью GPT). Кратко по каждому — что это и как использовать.

---

## Файлы

### 1. **cactus_cards_species.csv** (~1,2 МБ, ~3570 строк) — основной полезный файл

- **Что:** список видов кактусов с принятыми названиями (accepted_name, accepted_author), родом (accepted_genus, species), синонимами (all_names, all_names_with_status — через `|`), колонками для морфологии (morphology_stem, morphology_spines, morphology_flower, morphology_fruit) и заметок (notes_ru).
- **Источник:** идентификаторы WFO (World Flora Online), например `wfo-0000135353`.
- **Использование:** лучший кандидат для обогащения карточек на сайте (синонимы, морфология) и для сверки с нашим деревом. При слиянии проверять дубликаты и соответствие родов.

### 2. **cactus_wikipedia_list.csv** (~104 КБ, ~1550 строк)

- **Что:** список заголовков статей Википедии (number, title, pageid, length, touched). Смесь: роды и виды кактусов (Mammillaria, Carnegiea_gigantea), общие статьи (Cactus, Nopal), люди (Curt_Backeberg), места (Boyce_Thompson_Arboretum) и т.д.
- **Использование:** для сверки названий и поиска статей; не является чистым списком только видов — нужна фильтрация по смыслу (род/вид).

### 3. **cactus_only.csv** (~14 МБ, ~23 265 строк)

- **Что:** таблица в формате Darwin Core / TEN (taxonID, scientificName, family, subfamily, tribe, genus и т.д.).
- **Внимание:** не все строки — кактусы. В файле есть записи с **family = Brassicaceae** (и другие семейства). Строк с **Cactaceae** — около 115. Перед использованием **обязательно фильтровать по family = Cactaceae**, иначе попадут чужие семейства.

### 4. **sample_structure.csv** (~55 КБ, ~100 строк)

- **Что:** образец структуры таксономической таблицы (те же колонки, что в cactus_only).
- **Внимание:** данные в файле относятся к **Cyperaceae** (осоки), не к кактусам. Это шаблон формата, а не список кактусов. Для кактусов не использовать как источник видов.

### 5. **cactus.json** (~471 КБ)

- **Что:** JSON-ответ API Википедии — дамп одной статьи «Cactus» (внутри HTML-разметка страницы).
- **Использование:** текст общей статьи про семейство; не список видов. При необходимости можно парсить для текста, но не для таксономии.

---

## Рекомендации

- **Для сайта и синонимов:** в первую очередь использовать **cactus_cards_species.csv** (принятые названия + синонимы + заготовки под морфологию).
- **cactus_only.csv:** использовать только строки с `family = Cactaceae`; остальное отфильтровывать.
- **sample_structure.csv:** не считать источником по кактусам (данные по осокам).
- Папку можно оставить в `data/sources/cactus_database` (маленькими буквами). Если в корне проекта осталась старая папка `CACTUS_DATABASE`, её можно удалить вручную — актуальная копия здесь.

## Обогащение карточек на сайте

Скрипт **`scripts/enrich_species_from_cactus_cards.py`** подтягивает из **cactus_cards_species.csv** в наши виды (где совпадают род + эпитет):
- **nameHistory** — цепочка синонимов (из колонки all_names)
- **morphology_stem, morphology_spines, morphology_flower, morphology_fruit** — если в файле заполнены
- **photo_main_url, photo_flower_url, photo_spines_url** — ссылки на фото
- **notes_ru** — заметки

При этом **подвиды, разновидности, формы (infraspecific)** не изменяются и не удаляются. Дубликаты по полному названию не создаём; вариации (var., subsp., f.) сохраняются.

**Почему в карточке не было морфологии и фото:** в cactus_cards_species.csv колонки morphology_* и photo_* **пустые** (0 заполненных строк). Чтобы появились описание и картинки, используется скрипт **`scripts/fetch_wikipedia_morphology_photo.py`**: он запрашивает у Википедии (REST API) краткое описание и главное фото по латинскому названию вида и пишет их в `morphology_stem` и `photo_main_url`. Запуск: `python3 scripts/fetch_wikipedia_morphology_photo.py` (для всех видов, с паузой 1 сек — займёт время) или `python3 scripts/fetch_wikipedia_morphology_photo.py --limit 50` для теста на 50 видах.
