# Пошагово: как отправить изменения на сайт

Делай это **после** того, как в терминале скрипт написал что-то вроде «Готово. Переведено видов (полей): 50, обновлено родов: 12» (числа не нули). Тогда у тебя на компьютере изменены файлы — их нужно отправить на GitHub.

---

## Шаг 1. Открой терминал

Тот же терминал или новый — не важно. Главное, что ты в нём.

---

## Шаг 2. Перейди в папку проекта

Скопируй и вставь, нажми Enter:

```
cd /Users/alexanderermolovich/Documents/cactus
```

---

## Шаг 3. Посмотри, что изменилось

Вставь и Enter:

```
git status
```

Должны появиться списки файлов (modified: data/species/...). Если вместо этого написано **«nothing to commit, working tree clean»** — значит, ничего нового нет, на сайт отправлять нечего.

---

## Шаг 4. Добавить всё в коммит

Вставь и Enter:

```
git add -A
```

Ничего не напишет — это нормально.

---

## Шаг 5. Сделать коммит

Вставь и Enter:

```
git commit -m "Перевод морфологии на русский: ночной прогон"
```

Должно появиться что-то вроде «X files changed».

---

## Шаг 6. Отправить на сайт

Вставь и Enter:

```
git push origin main
```

Может спросить логин/пароль (или использовать сохранённый токен). Когда закончит — изменения уже на сайте.

---

## Если в конце скрипта было «Переведено: 0, обновлено родов: 0»

Значит, API перевода ничего не вернул (лимит или ошибка), файлы **не изменились**. Делать шаги 2–6 не нужно — коммитить нечего. Можно запустить скрипт перевода позже (например, на следующий день), когда лимит обновится.
