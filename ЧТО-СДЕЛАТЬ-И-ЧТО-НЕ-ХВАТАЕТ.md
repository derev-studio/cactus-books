## Что именно надо сделать (по шагам)

### 1. Загрузить папку media, чтобы в книге были картинки

Нужно, чтобы на GitHub в репозитории **derev-studio/cactus-books** появилась такая же структура, как у тебя на компе:

- В репо уже есть папка **cactus**, внутри неё — **книга-кактусология** с файлами `book-read.html`, `book-read-en.html`, `book-read-ru.html`.
- Нужно добавить туда же папку **media** со всеми картинками (image1.jpeg, image2.jpeg, …), то есть в итоге должно быть:
  - **cactus/книга-кактусология/book-read.html** (и остальные HTML книги)
  - **cactus/книга-кактусология/media/image1.jpeg**
  - **cactus/книга-кактусология/media/image2.jpeg**
  - и так далее для всех файлов из твоей папки `книга-кактусология/media/`.

**Как загрузить (два варианта).**

**Вариант А — через сайт GitHub (если папка media не слишком большая):**

1. Открой в браузере:  
   https://github.com/derev-studio/cactus-books  
2. Зайди в папку **cactus** → потом в **книга-кактусология**.
3. Нажми **«Add file»** → **«Upload files»**.
4. Открой на компе папку:  
   **Документы → cactus → книга-кактусология → media**  
   Перетащи в окно GitHub **все файлы из папки media** (image1.jpeg, image2.jpeg и т.д.).  
   Если GitHub не даёт залить всё сразу (много файлов), загружай по частям, например по 50–100 файлов за раз.
5. Внизу нажми **«Commit changes»**.

**Вариант Б — через Git (терминал на компе):**

1. Открой терминал (или Cursor → Terminal).
2. Перейди в папку проекта:
   ```bash
   cd /Users/alexanderermolovich/Documents/cactus
   ```
3. Проверь, что папка связана с репо Даши (если ты пушишь в derev-studio/cactus-books):
   ```bash
   git remote -v
   ```
   Если там другой репозиторий (твой), то push уйдёт в твой репо. Чтобы пушить в **derev-studio/cactus-books**, нужно:
   ```bash
   git remote set-url origin https://github.com/derev-studio/cactus-books.git
   ```
4. Добавь папку книги с media и закоммить:
   ```bash
   git add книга-кактусология/
   git status
   git commit -m "Добавить книгу Кактусология с папкой media (картинки)"
   git push origin main
   ```
   Если репозиторий устроен так, что сайт лежит в ветке **main**, а папка **cactus** — внутри репо, то в твоей локальной папке cactus должен быть не весь репо, а только её содержимое, и тогда эти команды нужно выполнять из корня репо, где есть папка **cactus**. Если у тебя на компе только папка cactus (без родительского cactus-books), то через Git залить именно в **cactus/книга-кактусология/** на GitHub можно только так: либо клонировать derev-studio/cactus-books, скопировать туда свою папку cactus (вместе с книга-кактусология/media), потом сделать add/commit/push из корня репо.

Кратко: **если удобнее через браузер — используй Вариант А.** Если привык к Git и репо уже клонирован — Вариант Б из корня cactus-books.
